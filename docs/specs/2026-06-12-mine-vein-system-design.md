# MoreOres 矿区系统设计规格 (v1.1.0)

> 粘液科技附属插件 — 区块矿脉系统 + 矿机 + 仓库 + 扫描器 + 钻头
>
> 日期: 2026-06-14 | 版本: 1.1.0

---

## 1. 概述

在 MoreOres 已有43种矿物和机器的基础上，新增完整的**区块矿脉系统**。该系统将Minecraft世界划分为区块，每个区块拥有独立的矿量数据，玩家通过矿脉扫描器探测、矿机开采、矿产仓库存储来获取矿物。

### 核心概念

| 概念 | 定义 |
|------|------|
| **岩上空间** | 基岩以上的部分（不含太空），即 Y ≥ world.getMinHeight() 以上 |
| **岩下空间** | 基岩以下的部分，即 Y < world.getMinHeight()（通常 Y < -64） |
| **矿脉** | 区块中蕴含的矿物总量，以数量表示 |
| **矿区** | 矿量较高的区块，4096个区块中约128个 |
| **矿物种类** | 由区块所在群系决定，详见 biome-ore-mapping.md |
| **矿余比 (r)** | r = 剩余矿量 / 总矿量，代表当前开采深度。r 从 1（浅）递减至 0（深） |
| **岩上空间矿石含量 (S)** | 每个区块随机生成的数值（15%~35%），表示岩上空间矿物占总量的比例 |

## 2. 子系统一：区块矿量系统 (MineVeinManager)

### 2.1 核心算法

每个区块的矿量由区块坐标 `(chunkX, chunkZ)` 作为伪随机种子确定，保证：
- 同一区块每次查询结果一致（确定性）
- 不同区块结果独立（随机性）

```java
long seed = world.getSeed() ^ (chunkX * 73856093L) ^ (chunkZ * 19349663L);
Random rng = new Random(seed);
```

### 2.2 矿脉等级

| 等级 | 矿量范围 | 概率 | 含义 |
|------|----------|------|------|
| 空 | 0 ~ 300 | 自动（非矿区） | 无商业价值，微量矿产 |
| 小 | 300 ~ 1,000 | 50%（矿区中） | 小型矿脉 |
| 中 | 1,000 ~ 15,000 | 30%（矿区中） | 中型矿脉 |
| 大 | 15,000 ~ 50,000 | 19.5%（矿区中） | 大型矿脉 |
| 超大 | 50,000+ | 0.5%（矿区中） | 超大型矿脉 |

### 2.3 矿区判定

1. 先判定是否为矿区：概率 = 128/4096 = 3.125%
2. 如果是矿区，再判定矿脉等级和具体矿量
3. 如果非矿区，矿量在 0~300 随机

```java
boolean isMiningZone = rng.nextDouble() < 0.03125;
if (isMiningZone) {
    double tierRoll = rng.nextDouble();
    if (tierRoll < 0.005)        // 超大 0.5%
        veinSize = 50000 + rng.nextInt(200000);
    else if (tierRoll < 0.02)    // 大 19.5%
        veinSize = 15000 + rng.nextInt(35000);
    else if (tierRoll < 0.32)    // 中 30%
        veinSize = 1000 + rng.nextInt(14000);
    else                         // 小 50%
        veinSize = 300 + rng.nextInt(700);
} else {
    veinSize = rng.nextInt(300); // 非矿区 0~300
}
```

### 2.4 岩上空间矿石含量 (S)

每个区块随机生成一个 **岩上空间矿石含量 S**（15%~35%），表示该区块岩上空间矿物占总矿量的比例。

**权重机制**：矿脉大小越大，S 值越倾向于取低值（大矿脉的矿物更多集中在岩下空间）。

```java
// 生成15~35之间的S值（百分比），加权：矿脉越大，低值权重越高
double baseRandom = rng.nextDouble(); // 0~1
double sizeWeight = Math.min(veinSize / 50000.0, 1.0); // 0~1，矿脉越大值越高
// 使用beta分布模拟：sizeWeight越大，偏向低值
double sRoll = baseRandom * (1.0 - sizeWeight * 0.5); // 偏向低值
double surfaceContent = 0.15 + sRoll * 0.20; // 15%~35%
```

**岩上空间矿物枯竭**：当 r ≤ (1 - S) 时，岩上空间矿物已全部开采完毕，矿机只能从岩下空间产出矿物。

### 2.5 矿物种类分配

**矿物种类由区块所在群系决定**，每个群系有固定矿物池。矿物按稀有度分为三级权重：

| 稀有度 | 权重 | 含义 |
|--------|------|------|
| 高 (H) | 40 | 常见矿物 |
| 中 (M) | 20 | 较常见至较稀有 |
| 低 (L) | 5 | 稀有至极稀有 |
| V | — | MC原版物品（石英/萤石），非MoreOres矿物 |

**所有区块（有矿脉/无矿脉）都使用相同的群系矿物池**，区别仅在于矿量大小。区块产出时，从群系矿物池中按权重随机选取 1~3 种矿物。

详细的群系-矿物-权重映射表见 [biome-ore-mapping.md](biome-ore-mapping.md)。

### 2.6 数据存储

矿量数据**无需文件持久化**，直接通过种子实时计算（确定性算法保证一致性）：

```java
private final Map<Long, ChunkVeinData> cache = new ConcurrentHashMap<>();

public ChunkVeinData getVeinData(World world, int chunkX, int chunkZ) {
    long key = chunkKey(world, chunkX, chunkZ);
    return cache.computeIfAbsent(key, k -> calculateVeinData(world, chunkX, chunkZ));
}
```

### 2.7 数据类

```java
public class ChunkVeinData {
    final int chunkX;
    final int chunkZ;
    final int totalVeinSize;           // 总矿量
    volatile int remainingVeinSize;    // 剩余矿量（开采后减少）
    final boolean isMiningZone;        // 是否矿区
    final String[] mineralIds;         // 1~3种矿物ID
    final double surfaceContent;       // 岩上空间矿石含量 S（15%~35%）
    final Biome biome;                 // 中心块群系
}
```

## 4. 子系统三：矿机 (ChunkMiner)

### 4.1 基本属性

| 属性 | 值 |
|------|-----|
| ID | `CHUNK_MINER` |
| 显示名 | &6矿机 |
| 材质 | BLAST_FURNACE |
| 合成台 | ENHANCED_CRAFTING_TABLE |
| 基类 | AContainer (Slimefun电力机器) |
| UI | 6行54格（大箱子） |

### 4.2 工作机制

1. 矿机放置在某个区块内
2. 获取当前区块的 `ChunkVeinData`
3. **所有区块都能通过矿机获取矿物**：
   - 有矿脉区块：矿量大（300~50,000+），产出丰富
   - 无矿脉区块：矿量小（0~300），产出稀少
   - **矿物种类由群系决定，两种区块一致**
4. **矿物从上往下挖掘**，r 代表当前开采深度（从1递减至0）
5. **一个矿机只能放一个钻头**（放入附件插槽，多个钻头时仅第一个生效）
6. **矿余比 r = remainingVeinSize / totalVeinSize**（不可告知玩家）

### 4.3 产出速率公式

```
速率 = 基础速率 × max(1/r, 8) × 钻头系数
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 基础速率 | 每个处理周期的基础产出 | 1 |
| 1/r | 矿余比倒数，r越小速率越高（越深产出越快） | 1 ~ ∞ |
| max(1/r, 8) | 最低下限为8，确保基本速率 | ≥ 8 |
| 钻头系数 | 由安装的钻头等级决定（§6.4） | 1.5 ~ 4.0 |

| r 值 | max(1/r, 8) | 效果 |
|:----:|:-----------:|------|
| 1.0 | 8 | 开始开采，基础8倍 |
| 0.5 | 8 | 下限保底 |
| 0.125 | 8 | 下限保底 |
| 0.1 | 10 | 开始加速 |
| 0.01 | 100 | 深层高速产出 |

### 4.4 耗电公式

```
耗电 = 基础速率 × max(1/r, 30) × 钻头系数
```

| r 值 | max(1/r, 30) | 效果 |
|:----:|:------------:|------|
| 1.0 | 30 | 最低耗电 |
| 0.5 | 30 | 最低耗电 |
| 0.033 | 30 | 最低耗电 |
| 0.01 | 100 | 高耗电 |

**仅消耗电力，不需要额外燃料。**

### 4.5 损耗公式

```
实际损耗系数 = 1 + (1 / 钻头等级)
```

| 钻头等级 | 损耗系数 | 含义 |
|:--------:|:--------:|------|
| 2 (钢) | 1.500 | 每产出1个矿物消耗1.5个矿量 |
| 3 (大马士革) | 1.333 | 每产出1个矿物消耗1.333个矿量 |
| 5 (强化合金) | 1.200 | 每产出1个矿物消耗1.2个矿量 |
| 7 (金刚石) | 1.143 | 每产出1个矿物消耗1.143个矿量 |

高等级钻头 = 更低损耗 = 更高效利用矿脉。

### 4.6 岩上/岩下空间产出

矿物从上往下开采：
- 当 r > (1 - S) 时：岩上空间和岩下空间同时产出
- 当 r ≤ (1 - S) 时：岩上空间矿物枯竭，仅岩下空间产出
- **仅强化金刚石钻头 + 冷却剂附件可进入岩下空间工作**

### 4.7 附件插槽系统

矿机有 **7个附件插槽**（UI中的①位置），可放入以下物品：

| 附件类型 | 说明 | 数量限制 |
|----------|------|:--------:|
| 钻头 (DrillBit) | 决定产出速率和损耗 | 仅1个生效 |
| 冷却剂 (Coolant) | 岩下空间必需品 | 可多组 |
| 探测器 (DrillProbe) | 显示精确矿脉信息 | 仅1个 |
| 钻头连接管 (ConnectionPipe) | 挖掘必需品，不消耗 | 按需数量 |

**钻头连接管需求公式**：

```
所需数量 = round(2 × (1 - r) / S)
```

- r 越小（越深）→ 所需数量越多
- S 越大（岩上含量高）→ 所需数量越少
- 连接管放入后**不被消耗**
- **如果当前 r 值所需数量超过插槽中实际数量，矿机停止工作**

### 4.8 剩余矿量耗尽

当区块的 `remainingVeinSize ≤ 0` 时，矿机停止工作并显示"矿脉已耗尽"。

## 6. 子系统五：钻头系统 (DrillBit)

### 6.1 矿余比 (r)

矿余比（r）是衡量区块矿脉剩余量的关键参数：

```
r = 剩余矿量 / 总矿量
```

- r 从 1（未开采/浅层）递减至 0（耗尽/深层）
- **r 代表开采深度**：矿物从上往下挖掘，r 越小深度越深
- **r 是矿脉的内部属性**，不可告知玩家
- r 影响产出速率、耗电、冷却剂消耗、连接管需求

### 6.2 钻头定义

每个钻头有两个核心属性：
- **钻头等级 (Tier)**：用于计算产出速率、耗电量和损耗
- **耐久 (Durability)**：可使用多久，用尽后钻头损坏

### 6.3 钻头种类

| 钻头 | 等级 | 耐久 | 说明 |
|------|:----:|:----:|------|
| 钢钻头 | 2 | 500 | 基础钻头 |
| 大马士革钻头 | 3 | 800 | 中级钻头 |
| 强化合金钻头 | 5 | 2000 | 高级钻头 |
| 金刚石钻头 | 7 | 800 | 精密钻头（低耐久高效率） |
| 强化金刚石钻头 | 7 | 3500 | 精密+高耐久，可搭配冷却剂进入岩下空间 |

### 6.4 钻头等级对矿机的影响

| 等级 | 钻头系数 | 损耗系数 | 说明 |
|:----:|:--------:|:--------:|------|
| 2 | 1.5 | 1.500 | 钢钻头 |
| 3 | 2.0 | 1.333 | 大马士革钻头 |
| 5 | 3.0 | 1.200 | 强化合金钻头 |
| 7 | 4.0 | 1.143 | 金刚石/强化金刚石钻头 |

### 6.5 耐久消耗

钻头耐久根据开采空间不同，消耗速率不同：

| 开采空间 | 耐久消耗/矿 | 说明 |
|----------|:----------:|------|
| 岩上空间 | 1 | 每产出1个矿物消耗1点耐久 |
| 岩下空间 | max(1/r, 3) | r越小消耗越快，最低3点/矿 |

### 6.6 钻头槽位

**一个矿机只能放一个钻头**（放入7个附件插槽中的任意一个），但一个矿脉可以放置多个矿机（多矿机竞争同一矿脉矿量）。如果插槽中有多个钻头，**仅第一个钻头生效**。

## 6A. 子系统五A：冷却剂 (Coolant)

### 6A.1 基本属性

| 属性 | 值 |
|------|-----|
| ID | `COOLANT` |
| 显示名 | &b冷却剂 |
| 材质 | WATER_BUCKET |
| 合成台 | ENHANCED_CRAFTING_TABLE |

### 6A.2 功能

冷却剂是**附件**，放入矿机的7个附件插槽中：
- **仅在岩下空间开采时启用**
- 消耗速率 = `1/r` 每秒（r为矿余比）
- r越小（矿脉越深），消耗越快
- 冷却剂耗尽后，矿机停止工作（即使有钻头和电力）
- **仅强化金刚石钻头搭配冷却剂可进入岩下空间工作**

### 6A.3 消耗速率对照表

| r (矿余比) | 冷却剂消耗速率 (单位/秒) | 说明 |
|:----------:|:------------------------:|------|
| 1.0 | 1 | 刚开始开采 |
| 0.5 | 2 | 开采一半 |
| 0.1 | 10 | 接近耗尽，消耗极快 |
| 0.05 | 20 | 几乎耗尽 |

### 6A.4 合成配方 (3×3)

```
冰      水桶      冰
水桶    青金石    水桶
冰      水桶      冰
```

## 6B. 子系统五B：钻头连接管 (ConnectionPipe)

### 6B.1 基本属性

| 属性 | 值 |
|------|-----|
| ID | `CONNECTION_PIPE` |
| 显示名 | &a钻头连接管 |
| 材质 | IRON_BARS |
| 合成台 | ENHANCED_CRAFTING_TABLE |

### 6B.2 功能

钻头连接管是**附件**，放入矿机的7个附件插槽中：
- **挖掘必需品**，没有足够连接管矿机无法工作
- **不被消耗**（放入后永久生效，直到被取出）
- 所需数量随开采深度增加而增加

### 6B.3 需求公式

```
所需数量 = round(2 × (1 - r) / S)
```

| 参数 | 说明 |
|------|------|
| r | 矿余比（当前开采深度） |
| S | 岩上空间矿石含量（15%~35%） |
| round() | 四舍五入到整数 |

**示例**（假设 S = 0.25）：

| r 值 | 所需连接管数量 |
|:----:|:--------------:|
| 1.0 | 0 |
| 0.75 | 2 |
| 0.5 | 4 |
| 0.25 | 6 |
| 0.0 | 8 |

### 6B.4 工作停止条件

当 `round(2 × (1 - r) / S)` > 插槽中实际连接管数量时，矿机停止工作。

### 6B.5 合成配方 (3×3)

```
铁锭    铁锭    铁锭
空      铁棍    空
铁锭    铁锭    铁锭
```

## 8. 数据流

```
玩家放置矿机
    ↓
MineVeinManager.getVeinData(chunkX, chunkZ)
    ↓
ChunkVeinData {
    totalVeinSize, remainingVeinSize,
    minerals[], biome,
    surfaceContent (S, 15%~35%)
}
    ↓
矿余比 r = remainingVeinSize / totalVeinSize（内部属性，不告知玩家）
    ↓
ChunkMiner 检查：
    1. 是否有钻头？（仅第一个生效）
    2. 冷却剂是否足够？（岩下空间时）
    3. 连接管是否足够？round(2*(1-r)/S) ≤ 实际数量
    ↓
ChunkMiner 计算产出：
    速率 = 基础速率 × max(1/r, 8) × 钻头系数
    耗电 = 基础速率 × max(1/r, 30) × 钻头系数
    损耗 = 1 + (1/钻头等级)
    ↓
每个处理周期：
    1. 检查 remainingVeinSize > 0
    2. 检查钻头耐久 > 0
    3. 检查冷却剂（岩下空间时）
    4. 检查连接管数量
    5. 计算产出量 = 速率
    6. 实际消耗矿量 = 产出量 × 损耗系数
    7. remainingVeinSize -= 实际消耗矿量
    8. 钻头耐久 -= (岩上:1 / 岩下:max(1/r,3))
    9. 冷却剂消耗 = 1/r 每秒（岩下空间时）
    10. 输出矿物到输出槽
    ↓
漏斗/货运自动取出 → 矿产仓库存储
```

## 11. 设计决策记录

1. **矿量确定性** — 用区块坐标+世界种子做随机种子
2. **无需持久化** — 种子算法确定性，内存缓存即可
3. **矿脉耗尽** — remainingVeinSize 递减，耗尽后矿机停止
4. **货运兼容** — 使用标准 Bukkit Inventory 接口
5. **卫星跳过** — 涉及外部addon依赖，暂不实现
6. **矿物种类群系决定** — 每个群系有固定矿物池
7. **三级稀有度权重** — 高(40)、中(20)、低(5)
8. **所有区块可开采** — 区别仅在矿量大小
9. **仅电力驱动** — 不需要额外燃料
10. **r代表深度** — 矿物从上往下挖掘，r递减代表深度增加
11. **r不可告知玩家** — 矿余比是内部属性
12. **岩上含量动态化** — 每个区块随机15%~35%，大矿脉倾向低值
13. **附件系统** — 冷却剂、探测器、连接管均为附件，放入7个插槽
14. **仅一个钻头生效** — 多个钻头时仅第一个生效
15. **钻头连接管** — 挖掘必需品，不消耗，数量随深度增加
16. **损耗与等级反比** — 高等级钻头 = 低损耗 = 高效利用矿脉
17. **耐久双模式** — 岩上1/矿，岩下max(1/r,3)/矿
