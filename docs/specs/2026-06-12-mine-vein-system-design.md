# MoreOres 矿区系统设计规格 (v0.1.0)

> 粘液科技附属插件 — 区块矿脉系统 + 矿机 + 仓库 + 扫描器 + 钻头
>
> 日期: 2026-06-12 | 版本: 1.0-final

---

## 1. 概述

在 MoreOres 已有43种矿物和机器的基础上，新增完整的**区块矿脉系统**。该系统将Minecraft世界划分为区块，每个区块拥有独立的矿量数据，玩家通过矿脉扫描器探测、矿机开采、矿产仓库存储来获取矿物。

### 核心概念

| 概念 | 定义 |
|------|------|
| **岩上空间** | 基岩以上的部分（不含太空），即 Y ≥ world.getMinHeight() 以上 |
| **岩下空间** | 基岩以下的部分，即 Y < world.getMinHeight()（通常 Y < -64） |
| **矿脉** | 区块中蕴含的矿物总量，以数量表示 |
| **矿区** | 矿量较高的区块，4096个区块中约128个 |
| **矿物种类** | 由区块所在群系决定，详见 biome-ore-mapping.md |

## 2. 子系统一：区块矿量系统 (MineVeinManager)

### 2.1 核心算法

每个区块的矿量由区块坐标 `(chunkX, chunkZ)` 作为伪随机种子确定，保证：
- 同一区块每次查询结果一致（确定性）
- 不同区块结果独立（随机性）

```java
long seed = world.getSeed() ^ (chunkX * 73856093L) ^ (chunkZ * 19349663L);
Random rng = new Random(seed);
```

### 2.2 矿脉等级

| 等级 | 矿量范围 | 概率 | 含义 |
|------|----------|------|------|
| 空 | 0 ~ 300 | 自动（非矿区） | 无商业价值，微量矿产 |
| 小 | 300 ~ 1,000 | 50%（矿区中） | 小型矿脉 |
| 中 | 1,000 ~ 15,000 | 30%（矿区中） | 中型矿脉 |
| 大 | 15,000 ~ 50,000 | 19.5%（矿区中） | 大型矿脉 |
| 超大 | 50,000+ | 0.5%（矿区中） | 超大型矿脉 |

### 2.3 矿区判定

1. 先判定是否为矿区：概率 = 128/4096 = 3.125%
2. 如果是矿区，再判定矿脉等级和具体矿量
3. 如果非矿区，矿量在 0~300 随机

```java
boolean isMiningZone = rng.nextDouble() < 0.03125;
if (isMiningZone) {
    double tierRoll = rng.nextDouble();
    if (tierRoll < 0.005)        // 超大 0.5%
        veinSize = 50000 + rng.nextInt(200000);
    else if (tierRoll < 0.02)    // 大 19.5%
        veinSize = 15000 + rng.nextInt(35000);
    else if (tierRoll < 0.32)    // 中 30%
        veinSize = 1000 + rng.nextInt(14000);
    else                         // 小 50%
        veinSize = 300 + rng.nextInt(700);
} else {
    veinSize = rng.nextInt(300); // 非矿区 0~300
}
```

### 2.4 岩上/岩下空间比例

| 环境 | 岩上空间占比 | 说明 |
|------|-------------|------|
| 矿区 | ~15% | 矿区中大部分矿在岩下 |
| 非矿区 | ~20% | 非矿区岩上占比稍高 |

```java
double surfaceRatio = isMiningZone ? 0.15 : 0.20;
int surfaceVein = (int)(veinSize * surfaceRatio);
int deepVein = veinSize - surfaceVein;
```

### 2.5 矿物种类分配

**矿物种类由区块所在群系决定**，每个群系有固定矿物池。矿物按稀有度分为三级权重：

| 稀有度 | 权重 | 含义 |
|--------|------|------|
| 高 (H) | 40 | 常见矿物 |
| 中 (M) | 20 | 较常见至较稀有 |
| 低 (L) | 5 | 稀有至极稀有 |
| V | — | MC原版物品（石英/萤石），非MoreOres矿物 |

**所有区块（有矿脉/无矿脉）都使用相同的群系矿物池**，区别仅在于矿量大小。区块产出时，从群系矿物池中按权重随机选取 1~3 种矿物。

```java
private SlimefunItemStack pickWeightedOre(List<WeightedOre> pool, Random rng) {
    int totalWeight = pool.stream().mapToInt(o -> o.rarity.weight).sum();
    int roll = rng.nextInt(totalWeight);
    for (WeightedOre ore : pool) {
        roll -= ore.rarity.weight;
        if (roll < 0) return ore.item;
    }
    return pool.get(pool.size() - 1).fallback;
}

int mineralCount = 1 + rng.nextInt(3);
List<SlimefunItemStack> minerals = pickMultipleWeighted(pool, mineralCount, rng);
```

详细的群系-矿物-权重映射表见 [biome-ore-mapping.md](biome-ore-mapping.md)。

### 2.6 数据存储

矿量数据**无需文件持久化**，直接通过种子实时计算（确定性算法保证一致性）：

```java
// 首次查询时计算，缓存到内存Map中
// 无需写入磁盘，重启后自动重新计算
private final Map<Long, ChunkVeinData> cache = new ConcurrentHashMap<>();

public ChunkVeinData getVeinData(World world, int chunkX, int chunkZ) {
    long key = chunkKey(world, chunkX, chunkZ);
    return cache.computeIfAbsent(key, k -> calculateVeinData(world, chunkX, chunkZ));
}
```

### 2.7 数据类

```java
public class ChunkVeinData {
    final int chunkX;
    final int chunkZ;
    final int totalVeinSize;
    volatile int remainingVeinSize;
    final boolean isMiningZone;
    final String[] mineralIds;
    final double surfaceRatio;
    final Biome biome;
}
```

### 3.3 数量估算规则

矿量四舍五入到最高位数，例如：
- 12,345 → "约 10,000"
- 567 → "约 600"
- 42 → "约 40"
- 3 → "约 0"

```java
private String estimateAmount(int amount) {
    if (amount == 0) return "0";
    int magnitude = (int) Math.pow(10, (int) Math.log10(amount));
    int rounded = Math.round((float) amount / magnitude) * magnitude;
    return "约 " + rounded;
}
```

## 4. 子系统三：矿机 (ChunkMiner)

### 4.1 基本属性

| 属性 | 值 |
|------|-----|
| ID | `CHUNK_MINER` |
| 显示名 | &6矿机 |
| 材质 | BLAST_FURNACE |
| 合成台 | ENHANCED_CRAFTING_TABLE |
| 基类 | AContainer (Slimefun电力机器) |
| UI | 3行27格，3输入+3输出 |

### 4.2 工作机制

1. 矿机放置在某个区块内
2. 获取当前区块的 `ChunkVeinData`
3. **所有区块都能通过矿机获取矿物**：
   - 有矿脉区块：矿量大（300~50,000+），产出丰富
   - 无矿脉区块：矿量小（0~300），产出稀少
   - **矿物种类由群系决定，两种区块一致**
4. **一个矿机只能放一个钻头**，但一个矿脉可以放置多个矿机（多矿机竞争同一矿脉矿量）
5. **矿余比 r = remainingVeinSize / totalVeinSize**
6. 根据以下公式计算每周期产出速率：

```
产出速率 (矿物/周期) = 基础速率 × 深度系数 × 钻头系数
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 基础速率 | 每个处理周期(10s)产出1个矿物 | 1 |
| 深度系数 | 矿机放置的Y坐标越低，系数越高 | 1.0 ~ 3.0 |
| 钻头系数 | 由安装的钻头等级决定（§6.4） | 1.5 ~ 4.0 |

7. **岩下空间限制**：
   - 所有钻头默认不能到达岩下空间
   - 仅强化金刚石钻头 + 岩下附件可开采岩下空间
   - 岩下空间开采需要冷却剂
   - **如果矿机无法挖掘岩下空间矿物，矿机停止工作**
   - **冷却剂耗尽后，矿机也停止工作**

### 4.3 深度系数计算

```java
double depthFactor = 1.0 + (double)(64 - minerY) / 64;
// Y=64 → 1.0, Y=0 → 2.0, Y=-64 → 3.0
// 低于-64时固定为3.0
if (minerY < -64) depthFactor = 3.0;
```

### 4.4 动态耗电

```
耗电 (J/s) = 基础耗电 × 深度系数 × 钻头系数
```

| 参数 | 值 |
|------|-----|
| 基础耗电 | 20 J/s (10 J/tick) |

**仅消耗电力，不需要额外燃料。**

### 4.5 采矿损耗

开采有损耗，并非矿有多少就能挖多少：
- 每次产出消耗的矿量 = 实际产出量 × 损耗系数
- 损耗系数 = 1.2（固定，即每产出1个矿物消耗1.2个矿量）

### 4.6 岩上/岩下空间产出

矿机在岩上空间和岩下空间都能开采矿物：

- 有矿脉区块：岩上空间产出约占 15%
- 无矿脉区块：岩上空间产出约占 20%

岩下空间（Y < world.getMinHeight()）需要**高级钻头**才能开采（按等级解锁），但产出更丰富（深度系数更高）。

### 4.7 剩余矿量耗尽

当区块的 `remainingVeinSize ≤ 0` 时，矿机停止工作并显示"矿脉已耗尽"。

### 4.8 UI 槽位

## 6. 子系统五：钻头系统 (DrillBit)

### 6.1 矿余比 (r)

矿余比（r）是衡量区块矿脉剩余量的关键参数：

```
r = 剩余矿量 / 总矿量
```

| r 值范围 | 含义 |
|----------|------|
| 1.0 | 未开采 |
| 0.5 | 开采一半 |
| 0.1 | 接近耗尽 |
| 0.0 | 耗尽 |

矿余比影响：
- 冷却剂消耗速率（岩下空间）
- 矿脉扫描器估算精度
- 矿机产出稳定性

### 6.2 钻头定义

每个钻头有两个核心属性：
- **钻头等级 (Tier)**：用于计算矿机产出速率和耗电量
- **耐久 (Durability)**：可使用多久，用尽后钻头损坏

### 6.3 钻头种类

| 钻头 | 等级 | 耐久 | 可达岩下空间 | 说明 |
|------|:----:|:----:|:------------:|------|
| 钢钻头 | 2 | 500 | ✗ | 基础钻头 |
| 大马士革钻头 | 3 | 800 | ✗ | 中级钻头 |
| 强化合金钻头 | 5 | 2000 | ✗ | 高级钻头 |
| 金刚石钻头 | 7 | 800 | ✗ | 精密钻头（低耐久高效率） |
| 强化金刚石钻头 | 7 | 3500 | ✗ | 精密+高耐久，可加附件 |

**注意：** 所有钻头默认均不能到达岩下空间。强化金刚石钻头可通过添加**岩下附件**获得岩下开采能力。

### 6.4 钻头等级对矿机的影响

| 参数 | 钻头等级影响 |
|------|-------------|
| 产出速率 | 钻头系数 = 1.0 + (tier - 1) × 0.5 |
| 耗电量 | 更高等级 = 更高耗电 |
| 耐久消耗 | 每次产出消耗1点耐久 |

**钻头等级与产出速率对照表：**

| 等级 | 钻头系数 | 说明 |
|:----:|:--------:|------|
| 2 | 1.5 | 钢钻头 |
| 3 | 2.0 | 大马士革钻头 |
| 5 | 3.0 | 强化合金钻头 |
| 7 | 4.0 | 金刚石/强化金刚石钻头 |

### 6.5 耐久系统

- 每次矿机产出矿物时，消耗1点钻头耐久
- 耐久耗尽后钻头自动损坏，矿机停止工作
- 玩家需要取出损坏钻头并放入新钻头
- 钻头耐久显示在物品lore中

### 6.6 矿机钻头槽位

**一个矿机只能放一个钻头**，但一个矿脉可以放置多个矿机（多矿机竞争同一矿脉矿量）。

钻头放入矿机输入槽的槽位10。

## 6B. 子系统五B：冷却剂 (Coolant)

### 6B.1 基本属性

| 属性 | 值 |
|------|-----|
| ID | `COOLANT` |
| 显示名 | &b冷却剂 |
| 材质 | WATER_BUCKET |
| 合成台 | ENHANCED_CRAFTING_TABLE |

### 6B.2 功能

冷却剂用于岩下空间采矿时的降温：
- **仅在岩下空间开采时启用**
- 消耗速率 = `1/r` 每秒（r为矿余比）
- r越小（矿脉越稀薄），消耗越快
- 冷却剂耗尽后，矿机停止工作（即使有钻头和电力）

### 6B.3 消耗速率对照表

| r (矿余比) | 冷却剂消耗速率 (单位/秒) | 说明 |
|:----------:|:------------------------:|------|
| 1.0 | 1 | 刚开始开采 |
| 0.5 | 2 | 开采一半 |
| 0.1 | 10 | 接近耗尽，消耗极快 |
| 0.05 | 20 | 几乎耗尽 |

### 6B.4 矿机冷却剂槽位

冷却剂放入矿机输入槽的槽位11（与钻头槽位相邻）。

### 6B.5 合成配方 (3×3)

```

```

### 6B.6 数学推导

冷却剂总消耗量（从r=1到r=r_end）：

```
总消耗 = ∫(r_start → r_end) (1/r) dr = ln(r_start) - ln(r_end)
```

示例：从r=1开采到r=0.1，总消耗 = ln(1) - ln(0.1) = 2.303单位冷却剂

## 8. 数据流

```
玩家放置矿机
    ↓
MineVeinManager.getVeinData(chunkX, chunkZ)
    ↓
ChunkVeinData { totalVeinSize, remainingVeinSize, minerals[], biome, ... }
    ↓
矿余比 r = remainingVeinSize / totalVeinSize
    ↓
ChunkMiner 检查：
    1. 是否有钻头？
    2. 钻头是否能到达当前开采深度？
    3. 岩下空间：是否有岩下附件+冷却剂？
    4. 冷却剂是否足够？
    ↓
ChunkMiner 根据 深度×钻头 计算产出速率
    ↓
每个处理周期：
    1. 检查 remainingVeinSize > 0
    2. 检查钻头耐久 > 0
    3. 检查冷却剂（岩下空间时）
    4. 计算产出量 = baseRate × depthFactor × drillFactor
    5. 消耗矿量 = 产出量 × 1.2（损耗）
    6. remainingVeinSize -= 消耗矿量
    7. 钻头耐久 -= 1
    8. 冷却剂消耗 = 1/r 每秒（岩下空间时）
    9. 输出矿物到输出槽
    ↓
漏斗/货运自动取出 → 矿产仓库存储
```

## 11. 设计决策记录

1. **矿量确定性** — 用区块坐标+世界种子做随机种子，保证同一区块每次查询结果一致
2. **无需持久化** — 种子算法确定性，重启后自动重新计算，内存缓存即可
3. **矿脉耗尽** — remainingVeinSize 递减，耗尽后矿机停止
4. **货运兼容** — 使用标准 Bukkit Inventory 接口，天然兼容漏斗和粘液货运
5. **损耗机制** — 1.2倍损耗系数，避免玩家无限制获取矿物
6. **卫星跳过** — 涉及外部addon依赖，暂不实现
7. **矿物种类群系决定** — 每个群系有固定矿物池，而非全局随机
8. **三级稀有度权重** — 高(40)、中(20)、低(5)，加权随机选取
9. **所有区块可开采** — 有矿脉/无矿脉区块都可用矿机，区别仅在矿量大小
10. **仅电力驱动** — 矿机不需要额外燃料
11. **钻头等级 = 产出效率** — 等级越高，钻头系数越大（1.5~4.0）
12. **钻头耐久** — 每次产出消耗1点耐久，耗尽后损坏
13. **一矿机一钻头** — 但一矿脉可放多矿机（竞争矿量）
14. **岩下空间需附件** — 仅强化金刚石钻头+附件可到达岩下空间
15. **冷却剂机制** — 岩下空间开采需冷却剂，消耗速率 1/r（矿余比越小消耗越快）
16. **矿余比 r** — 核心参数，影响冷却剂消耗和产出稳定性
